# 🛡️ InfoSec Vault: Glossary

These are the core terms every security professional needs to know. 

| Term | Definition | Why it matters in CS-205 |
| :--- | :--- | :--- |
| **Input Validation** | Checking data *before* a program processes it. | Prevents "Garbage In, Garbage Out" and security crashes. |
| **Logic Error** | A bug where the code runs, but does the wrong thing. | A logic error in a login screen can let the wrong person in. |
| **Encryption** | Scrambling data so only authorized users can read it. | Protecting the "Variables" we store in our programs. |
| **Syntax Error** | A "grammar" mistake in your code. | The computer won't even start the program if this exists! |
# 🛡️ InfoSec Glossary

| Term | Definition |
| :--- | :--- |
| **Variable** | A storage location (box) for data. |
| **String** | A sequence of characters (text). |
| **Integer** | A whole number. |
| **Assignment** | Using `=` to store a value in a variable. |
| **Input Validation** | Checking if data is safe/correct before using it. |
